# Заметки для разработчика

## Быстрая шпаргалка

### Перед началом работы
- [ ] Проверить текущий статус в CHANGELOG.md
- [ ] Ознакомиться с разделом [Unreleased]
- [ ] Проверить TODO список

### После изменений
- [ ] ✅ Обновить CHANGELOG.md (обязательно!)
- [ ] ✅ Обновить версию, если требуется
- [ ] ✅ Запустить тесты
- [ ] ✅ Проверить компиляцию
- [ ] ✅ Обновить документацию, если нужно

### Типы изменений для CHANGELOG

**Всегда записывать:**
- Новые функции/файлы
- Изменения существующих функций
- Исправления багов
- Изменения конфигурации
- Значимые изменения документации
- Добавление/удаление зависимостей

**Можно пропустить:**
- Мелкие опечатки
- Рефакторинг без изменения функциональности (можно кратко упомянуть)

### Стандартные команды

```bash
# Компиляция
npm run compile

# Тесты
npm run test

# Деплой
npm run deploy

# Локальный нод
npm run node
```

### Структура CHANGELOG

Раздел [Unreleased] используется для:
- Текущих изменений в разработке
- Планируемых улучшений
- Быстрых заметок о проделанной работе

При выпуске новой версии:
1. Перенести записи из [Unreleased] в новую версию
2. Добавить дату
3. Очистить [Unreleased] для новых записей
