# Быстрая настройка Resend для отправки email

## Шаг 1: Регистрация на Resend

1. Перейдите на https://resend.com
2. Нажмите **"Get Started"** или **"Sign Up"**
3. Зарегистрируйтесь (можно через GitHub, Google или email)
4. Подтвердите email (если нужно)

## Шаг 2: Получение API ключа

1. После входа в аккаунт, перейдите по ссылке: https://resend.com/api-keys
   ИЛИ
   - Нажмите на ваш аватар/профиль (верхний правый угол)
   - Выберите **"API Keys"** из меню

2. На странице API Keys:
   - Нажмите кнопку **"Create API Key"** (или "Add API Key")
   - Введите название (например: "El Toro Negro Dev")
   - Выберите разрешения:
     - Можно выбрать **"Full Access"** для тестирования
     - Или **"Sending Access"** (только отправка писем)
   - Нажмите **"Add"** или **"Create"**

3. **ВАЖНО:** Скопируйте API ключ сразу! Он показывается только один раз.
   - API ключ выглядит так: `re_123456789abcdefghijklmnopqrstuvwxyz`
   - Начинается с `re_`

## Шаг 3: Добавление ключа в проект

1. Откройте файл `frontend/.env.local` (создайте, если его нет)

2. Добавьте строку:
   ```env
   RESEND_API_KEY=re_ваш_ключ_здесь
   ```
   
   Например:
   ```env
   RESEND_API_KEY=re_abc123xyz789def456ghi012jkl345mno678pqr901
   ```

3. Сохраните файл

## Шаг 4: Перезапуск сервера

1. Остановите текущий сервер (Ctrl+C в терминале)
2. Запустите снова:
   ```bash
   npm run dev
   ```

## Шаг 5: Тестирование

1. Откройте Dashboard в браузере
2. Нажмите "Send Verification Email"
3. Проверьте ваш email (включая папку "Спам")
4. Письмо должно прийти в течение нескольких секунд

## Важные заметки

- **Бесплатный лимит**: 100 писем в день, 3000 в месяц
- **Отправитель по умолчанию**: `onboarding@resend.dev` (для теста)
- **Для production**: Нужно верифицировать свой домен в Resend

## Если что-то не работает

1. Проверьте, что API ключ скопирован полностью (начинается с `re_`)
2. Убедитесь, что файл `.env.local` находится в папке `frontend/`
3. Убедитесь, что сервер перезапущен после добавления ключа
4. Проверьте консоль сервера на наличие ошибок
