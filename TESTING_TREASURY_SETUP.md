# Настройка адреса казначейства для тестирования

## Быстрое решение

Для тестирования используйте адрес вашего собственного кошелька MetaMask.

### Шаг 1: Скопируйте адрес вашего кошелька

1. Откройте MetaMask
2. Нажмите на адрес кошелька (например, `0xa92d...d226`)
3. Адрес скопируется автоматически, или нажмите "Copy address"
4. Вы получите полный адрес, например: `0xa92d1234567890abcdef1234567890abcdefd226`

### Шаг 2: Создайте файл .env.local

1. Создайте файл `frontend/.env.local` в папке проекта
2. Добавьте туда:

```env
NEXT_PUBLIC_TREASURY_ADDRESS=0xa92d1234567890abcdef1234567890abcdefd226
```

**Замените адрес на адрес вашего кошелька!**

### Шаг 3: Перезапустите сервер

Переменные окружения загружаются только при запуске сервера, поэтому нужно перезапустить:

1. Остановите текущий сервер (Ctrl+C в терминале)
2. Запустите снова: `npm run dev`

### Шаг 4: Обновите страницу в браузере

После перезапуска сервера обновите страницу (F5 или Ctrl+R) и попробуйте снова.

---

## Варианты адресов для тестирования

### Вариант 1: Ваш собственный адрес (Рекомендуется для тестирования)

Используйте адрес вашего тестового кошелька MetaMask. Все ETH, которые вы отправите, будут приходить на этот адрес.

**Плюсы:**
- Просто настроить
- Вы можете видеть все транзакции в своем кошельке
- Удобно для тестирования

### Вариант 2: Другой тестовый адрес

Можете использовать любой другой адрес, если нужно отправить средства на другой кошелек.

### Вариант 3: Производственный адрес (только для production)

Для продакшена используйте адрес специального кошелька казначейства, который вы контролируете.

---

## Важно!

- ⚠️ **Для тестирования** используйте тестовые сети (Sepolia) и тестовые ETH
- ⚠️ **Не используйте** реальные средства на mainnet для разработки
- ✅ После добавления `.env.local` **обязательно перезапустите** dev server
- ✅ Файл `.env.local` уже в `.gitignore`, так что он не попадет в git

---

## Проверка

После настройки и перезапуска сервера:
1. Откройте консоль браузера (F12)
2. Проверьте, что ошибка "Treasury address not configured" исчезла
3. Попробуйте купить токены - должно работать!
